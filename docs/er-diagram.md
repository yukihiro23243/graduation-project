### 本サービスの概要（700文字以内）
サウナ好きがととのい体験・体感したこだわりを記録できるアプリケーションです。 行ったサウナ施設の情報や感想、満足度を記録できます。 自分だけのサウナ履歴コレクションとしても、他ユーザーとの情報共有にも使えます。
自分自身サウナに行くのが好きで、「どの施設が良かったか」「何度目でどんな整い方をしたか」を記録したいと思ったのがアプリケーション作成のきっかけで、自身と重ね、昨今のブームに乗りサウナへ興味を持ち始めた20代男性をメインターゲットとします。 隙間時間にスマートフォンが当たり前となっている現代の若者に対し、サウナ×アプリケーションの掛け算でターゲットを絞りました。
既存の競合サービスでは簡易的な記録とフリーメモのような機能となっていることが多いと感じ、文章では自由度がありつつも活字離れなど囁かれる現代で自身の体験を長々と綴るには億劫と感じる若者も多いと思うので、この課題へのアプローチを重点的に考えるアプリケーションとして目的を設定しました。
MVPで作る機能は下記「READMEに記載した機能」欄を参照とし、本リリースまでを見据えたとした機能としてはコメント関連機能やいいね機能、称号・タグ機能を想定しています。

### MVPで実装する予定の機能
- ユーザー登録機能
- ログイン機能
- メールアドレス変更機能
- パスワード変更機能
- 記録投稿機能（画像投稿含む）
- 記事閲覧機能（未ログインでも閲覧可能）
- 記事編集機能
- 記事削除機能

### テーブル詳細
#### Users(ユーザー情報)
- name : string / ユーザーの表示名 / ユニーク制約(山田太郎)
- email : string / ログイン用メールアドレス / ユニーク制約(taro@example.com)
- password : string / ログイン用パスワード
- created_at : datetime / 作られた日時
- updated_at : datetime / 最後に更新された日時

#### Posts(サウナ記録投稿)
- sauna_name : string / サウナ施設名(〇〇温泉)
- totonoi_score : integer / 整い度スコア
- content : text / フリーメモ
- image_url : string / 画像リンク
- created_at : datetime / 作られた日時
- updated_at : datetime / 最後に更新された日時

#### Comments(コメント欄)
- content : text / コメント本文
- created_at : datetime / 作られた日時
- updated_at : datetime / 最後に更新された日時

#### Likes(いいね欄)
- created_at : datetime / いいねした日時

#### Tags(タグ)
- name : string / タグ名(#サウナ最高、#水風呂キンキン)

#### Badges(称号)
- name : string / 称号名(10施設制覇、5都道県遠征)
