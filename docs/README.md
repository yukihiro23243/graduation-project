## サービス概要
サウナ好きがととのい体験・体感したこだわりを記録できるアプリケーションです。
行ったサウナ施設の情報や感想、満足度を記録できます。
自分だけのサウナ履歴コレクションとしても、他ユーザーとの情報共有にも使えます。


## このサービスへの思い・作りたい理由
自分自身サウナに行くのが好きで、「どの施設が良かったか」「何度目でどんな整い方をしたか」を記録したいと思ったのがきっかけです。
今までは時間が経つとどんなサウナだったか忘れてしまうこともあったので、記録として残せて後から見返せるアプリケーションがあれば良いなと思いました。


## 想定しているユーザー
自身と重ね、昨今のブームに乗りサウナへ興味を持ち始めた20代男性をメインターゲットとします。
隙間時間にスマートフォンが当たり前となっている現代の若者に対し、サウナ×アプリケーションの掛け算でターゲットを絞りました。


## 既存サービス・競合との差別化と課題へのアプローチについて
- サービス名： サウナイキタイ　~全国のサウナのサ活~
- URL： https://sauna-ikitai.com/posts
- どんなことができるか： 施設名と簡易的な温度等記録と他フリーメモ

例として挙げたこのサービスに限らず、既存アプリケーションでは簡易的な記録とフリーメモのような機能となっていることが多いと感じます。
文章では自由度がありつつも活字離れなど囁かれる現代では、自身の体験を長々と綴るには億劫と感じる若者も多いはずです。そこで、「爽快にサクッと記録」＆「こだわりと詳細を深く」の両立と最適な塩梅を目指し、ノンストレスでタイパ〇な体験を提供します。
フリーメモ欄ではなく、管理可能な設定タイトルと本文を自由に設定できる欄へ「備品のメーカー」「シャワーの水圧」など記録を残しておける機能を考えており、その点の独自性は推しポイントです。


## サービスの利用イメージ
サウナを利用した直後であれば、休憩場で充実感を感じながらリラックスしている若者も多く見かけます。
「サウナ履歴帳」としての活用で、そのタイミングだからこその小さな気づきも含め記録を残したいと思われるような利用イメージを想像しています。
そのほか行く前には今日どこのサウナに行こうかと見返しながら検討ができたり、他のユーザーの投稿を見て新しい施設を発見する楽しみも得られます。


## 利用を続けてもらうための工夫
ユーザーは使い続けることで自身これまでの記録がコレクションとなり、「このサウナにはあの時期に行ったな...」「このサウナには行ってなかった。」「遠くのサウナまで遠征してみよう。」などサウナライフの広がりを感じ、中毒性を与えることで長く愛されるサービスとなるよう運営します。
本リリース時に称号機能の実装を想定し、記録件数や訪れた施設数などに応じ各ユーザーへ称号を与えることで飽きが生まれづらいよう工夫します。


## 機能候補
### MVPで作る機能
- ユーザー登録
- ログイン機能
- メールアドレス変更機能
- パスワード変更機能
- 記録投稿機能（画像投稿含む）
- 記事閲覧機能（未ログインでも閲覧可能）
- 記事編集機能
- 記事削除機能

MVPでは、「こだわりと詳細を深く記録可能」としつつも視覚的に煩わしくないUI等、直感的で爽快な記録体験を目指すアプリケーションとしているため、この点を具体的に検証します。
詳細な記録も残せるというイメージが強くなり画面上は複雑となってしまうような懸念にならないよう、既存サービスとの比較・競合に限らず広くユーザー体験優れたデザインを参考として調整を考えています。


### 本リリースで作る機能
- コメント投稿機能
- コメント閲覧機能（未ログインでも閲覧可能）
- コメント編集機能
- コメント削除機能
- いいね機能
- いいね解除機能
- 称号機能
- タグ機能（#今日のととのい、#水風呂最高など）
- 地図API連携
- 整い統計ページ


## 使用する技術スタック
- 使用するフレームワーク
  Ruby on Rails 7.x
- データベース
  MySQL 8.x
- デプロイ先
  Render
- 使用予定のGem
  Devise：ユーザー認証機能（ログイン・ログアウト・登録）を実装。
　Active Storage：画像アップロード機能（サウナ施設写真など）に使用。
　Ransack（または簡易検索機能実装）：投稿の検索・ソート機能に利用予定。
　Chartkick + Groupdate：整い度の統計グラフ表示を実装予定。
　Tailwind CSS：デザインを効率的に整えるためのCSSフレームワーク。
　dotenv-rails：APIキーや環境変数を安全に管理。
